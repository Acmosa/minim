minim.qobuz.Session#

class Session(*, email: str | None = None, password: str | None = None, auth_token: str | None = None, user_agent: str | None = None)[source]#

Bases: object

A Qobuz API session.

Parameters:
emailstr, keyword-only, optional

Account email address. If it is not stored as QOBUZ_EMAIL in the operating system’s environment variables, it can be provided here.

passwordstr, keyword-only, optional

Account password. If it is not stored as QOBUZ_PASSWORD in the operating system’s environment variables, it can be provided here.

auth_tokenstr, keyword-only, optional

User authentication token. If it is not stored as QOBUZ_USER_AUTH_TOKEN in the operating system’s environment variables, it can be provided here.

user_agentstr, keyword-only, optional

User agent information to send in the header of HTTP requests.

Attributes:
API_URLstr

URL for the Qobuz API.

WEB_URLstr

URL for the Qobuz web player.

get_streams(id: int | str, type: str, *, quality: int | str = 27, save: bool = False, path: str | None = None, folder: bool = False, metadata: bool = True) → None | list[bytes][source]#

Get audio stream data for all tracks in an album or a playlist.

User authentication

Requires user authentication and an active Qobuz streaming plan for full, high-quality audio streams.

Note

This method is provided for convenience and is not a Qobuz API endpoint.

Parameters:
idint or str

Qobuz collection ID.

typestr

Qobuz collection type.

Valid values: "album" and "playlist".

qualitystr, keyword-only, default: 27

Maximum audio quality.

Valid values:

  • 5 or "LOSSY" for constant bitrate (320 kbps) MP3.

  • 6 or "CD" for CD-quality (16-bit, 44.1 kHz) FLAC.

  • 7 or "HI-RES-96" for up to 24-bit, 96 kHz Hi-Res FLAC.

  • 27 or "HI-RES-192" for up to 24-bit, 192 kHz Hi-Res FLAC.

savebool, keyword-only, default: False

Determines whether the streams are saved to audio files.

pathstr, keyword-only, optional

If save=True, path in which the audio files are saved.

folderbool, keyword-only, default: False

Determines whether a folder in path (or in the current directory if path is not specified) is created to hold the audio files.

metadatabool, keyword-only, default: True

Determines whether the audio files’ metadata is populated.

Returns:
streamslist

Audio stream data. If save=True, None is returned and the streams are saved to audio files instead.
